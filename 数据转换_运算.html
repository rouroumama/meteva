    将站点数据赋值到统一站表上

    put_stadata_on_station(sta,station)
    在检验中经常需要采用统一的站点表，而实时读入的站点数据在站点的数量和排列顺序上可能每次有有所不同。本函数的功能就是将站点数据规范成具有相同站点列表和排列顺序的数据。

    参数 说明
    sta 站点数据
    station 站点数据,站点数据,其中id，lon,lat列被采用
    return 站点数据，站点数据，id,lon,lat和station中一致，且排列顺序一致。level,time,dtime则采用sta中相同id对应的值，数据列中如果某个id在sta中有对应行，则采用sta的数值，否则采用station中的数值。

    调用示例

    复制sta1 = meb.comp.put_stadata_on_station(sta,station)
print(sta1)

    mutiply_on_level_time_dtime_id(sta1,sta2,how = “left”,default = None)

    站点数据取较大值

    max_on_level_time_dtime_id(sta1,sta2,how = “left”,default = None)
    将sta1,sta2基于level、time、dtime、id坐标对齐后再取两者较大值

    站点数据取较小值

    min_on_level_time_dtime_id(sta1,sta2,how = “left”,default = None)
    将sta1,sta2基于level、time、dtime、id坐标对齐后再取两者较小值

    参数 说明
    sta1 站点数据
    sta2 站点数据
    how how = “left”时返回的数据表和sta1一致，how = “right”时返回的数据表和 sta2一致，how = “inner”时，返回结果是两个数据表的交集，how = “outer”时返回结果是两个数据表的并集
    default 当返回结果中的某些站点在sta1,sta2中缺失了，就讲缺失的部分先赋值成defalut后再运算
    return 站点数据，站点数据运行后的结果

    网格数据滑动平均

    moving_ave(grd, half_window_size)
    求网格数据的水平方向的正方形窗口滑动平均场。 该函数是FSS检验的基础函数，它同样可以基于确定性预报的邻域法概论产品的制作当中。

    参数 说明
    grd 网格数据
    half_window_size 半边窗口的网格数据，例如当half_window_size = 1,时，计算区域中部的一个网格点的平均用到 3×3个格点，当half_window_size = 2时 用到了5×5个格点求平均, 在区域的边缘，计算平均的格点数为窗口内实际格点数，每个位置会略有差异，但会正确保持平均值的含义
    return 平滑后的网格数据

    调用示例

    网格数据滑动标准差

    moving_std(grd, half_window_size)
    求网格数据的水平方向的正方形窗口滑动标准差。

    参数 说明
    grd 网格数据
    half_window_size 半边窗口的网格数据，例如当half_window_size = 1,时，计算区域中部的一个网格点的平均用到 3×3个格点，当half_window_size = 2时 用到了5×5个格点求平均, 在区域的边缘，计算平均的格点数为窗口内实际格点数，每个位置会略有差异，但会正确保持平均值的含义
    return 滑动标准差场，每个格点是起附近窗口内原始场的标准差

    调用示例

    复制v_std = meb.moving_std(rain24,half_window_size=5)  # 平滑操作不改变网格范围和网格距
print(time.time() - start)
meb.tool.plot_tools.contourf_2d_grid(v_std,cmap =meb.cmaps.rain_24h,title = "滑动窗口的标准差",dpi = 100)
